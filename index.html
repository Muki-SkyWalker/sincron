Scalax

The missing Scala standard library. The purpose of Scalax is to build missing interfaces and functionalities that should be standard and to (hopefully) push these as SLIPs.

The name scalax mirrors the javax package for Java and in javax that's where extensions to an existing JRE go.

Usage

Scalax is being split in small and reusable packages. Some will provide a Scala.js / Javascript compatible implementation (within reason) for helping with cross compiled code.

Atomic References (scalax-atomic)

See Atomic References in the Wiki.

Dependency for the JVM:

libraryDependencies += "org.monifu" %% "scalax-atomic" % "0.3"

For Scala.js / Javascript or for cross-compiled projects:

libraryDependencies += "org.monifu" %%% "scalax-atomic" % "0.3"
